What Are Bookmarklets?

A bookmarklet is a small JavaScript program stored as a bookmark in your browser. When clicked, it performs a specific function on the current webpage. Unlike browser extensions, bookmarklets don't require installation, don't need permissions, and work across different browsers.

No Installation Required

Bookmarklets are stored as bookmarks in your browser. Just drag and drop to add them, no installation process needed.

Privacy Friendly

Bookmarklets run only when you click them. They don't run in the background or collect data without your knowledge.

Cross-Browser Compatible

Most bookmarklets work across Chrome, Firefox, Safari, Edge, and other modern browsers without modification.

Lightweight

Bookmarklets are small JavaScript snippets that don't slow down your browser or consume system resources.

Open Source

All bookmarklets in our collection are open source. You can view and understand exactly what they do.

Portable

Bookmarklets can be easily shared, exported, and imported between browsers and devices.

How Bookmarklets Differ from Extensions

Bookmarklets don't require installation through browser stores, don't need special permissions, run only when clicked, and work across different browsers. Extensions are installed, can run automatically, and are usually browser-specific.

When to Use Bookmarklets

Bookmarklets are perfect for quick tasks, one-time actions, tools you use occasionally, situations where you can't install extensions, and when you want to avoid browser bloat.

Installation Guide

Installing bookmarklets is simple and takes just a few seconds. Follow the steps below to get started.

Step 1: Show Your Bookmarks Bar

First, make sure your bookmarks bar is visible. Press Ctrl+Shift+B (Windows/Linux) or Cmd+Shift+B (Mac) to toggle the bookmarks bar. You should see a bar below your address bar.

Step 2: Find the Bookmarklet

Navigate to our Browse page and find the bookmarklet you want to install. Each bookmarklet has a link or button you can use.

Step 3: Install the Bookmarklet

Method 1 (Easiest): Simply drag the bookmarklet link from the page to your bookmarks bar. Method 2: Right-click on the bookmarklet link and select "Bookmark this link" or "Add to bookmarks".

Step 4: Verify Installation

You should see the bookmarklet appear in your bookmarks bar. To test it, navigate to any webpage and click the bookmarklet. Make sure you're on a webpage first, as bookmarklets need a page to run on.

Browser-Specific Instructions

Google Chrome

Enable the bookmarks bar with Ctrl+Shift+B (Windows/Linux) or Cmd+Shift+B (Mac). Drag the bookmarklet link to the bar, or right-click and select "Bookmark".

Mozilla Firefox

Right-click the bookmarklet link and select "Bookmark This Link", then choose where to save it. You can also drag it to the bookmarks toolbar. Enable the toolbar via View → Toolbars → Bookmarks Toolbar.

Safari

Enable the bookmarks bar via View → Show Bookmarks Bar or Cmd+Shift+B. Drag the bookmarklet link to the bar, or right-click and select "Add Bookmark".

Microsoft Edge

Similar to Chrome. Enable the bookmarks bar with Ctrl+Shift+B and drag the link, or right-click and select "Add to favorites". The bookmarks bar will appear below the address bar.

Opera

Enable the bookmarks bar with Ctrl+Shift+B (Windows/Linux) or Cmd+Shift+B (Mac). Drag the bookmarklet link to the bar, or right-click and select "Bookmark page". Opera's bookmark system works similarly to Chrome.

Brave Browser

Brave is Chromium-based, so it works like Chrome. Enable the bookmarks bar with Ctrl+Shift+B (Windows/Linux) or Cmd+Shift+B (Mac). Drag the bookmarklet link to the bar, or right-click and select "Bookmark".

Using Bookmarklets

Master the art of using bookmarklets with these practical tips and techniques for maximum effectiveness.

Timing Matters

Some bookmarklets work best when run at specific times. For dynamic websites, wait for the page to fully load. For Single Page Applications (SPAs), you may need to run bookmarklets after navigation completes.

Workflow Integration

Integrate bookmarklets into your daily workflow. Create keyboard shortcuts for frequently used bookmarklets using browser extensions, or organize them in folders by task type (editing, viewing, testing).

Combining Bookmarklets

Some bookmarklets work great in sequence. For example, run a "remove styles" bookmarklet before a "readability" one. Experiment with combinations, but test on non-critical pages first.

Site-Specific Usage

Some bookmarklets are designed for specific sites or site types. Check the bookmarklet description for compatibility notes. Social media bookmarklets may only work on certain platforms.

Undo and Recovery

Most bookmarklets that modify page content can be undone by refreshing the page. For bookmarklets that make permanent changes, check if they provide an undo function or save a backup first.

Customization Tips

You can view the source code of any bookmarklet to understand how it works. See our Advanced Topics for more information on how bookmarklets work.

Troubleshooting

If you're experiencing issues with a bookmarklet, try these troubleshooting steps before reporting a problem.

Bookmarklet Not Working

Make sure you're on a webpage (bookmarklets need a page to run on). Check if the bookmarklet is compatible with the website. Try refreshing the page and running the bookmarklet again.

Check Browser Console

Open your browser's developer tools (F12) and check the console for error messages. This can help identify what's going wrong. Look for JavaScript errors or warnings related to the bookmarklet.

JavaScript Enabled

Ensure JavaScript is enabled in your browser settings. Bookmarklets require JavaScript to function. Check your browser's security settings to make sure JavaScript isn't blocked.

Security Restrictions

Some websites have security restrictions that prevent bookmarklets from running. This is common on banking sites, government sites, and other sensitive pages. The bookmarklet may not work on these sites for security reasons.

Reinstall Bookmarklet

If a bookmarklet stops working, try removing it and reinstalling it. Sometimes bookmarklets can become corrupted or outdated. Get the latest version from our Browse page.

Browser Compatibility

Check if the bookmarklet is compatible with your browser version. Some bookmarklets may have browser-specific behavior. We recommend using the latest version of your browser for the best experience.

Basics & Getting Started

What is a bookmarklet?

A bookmarklet is a small JavaScript program stored as a bookmark in your browser. When clicked, it performs a specific function on the current webpage. Unlike browser extensions, bookmarklets don't require installation, don't need special permissions, and work across different browsers.

For more detailed information about what bookmarklets are and how they work, see our What Are Bookmarklets? section.

How are bookmarklets different from browser extensions?

Bookmarklets and browser extensions serve similar purposes but have key differences:

  • Installation: Bookmarklets don't require installation through browser stores - just add them to your bookmarks. Extensions need to be installed through the browser's extension store and may require approval.
  • Permissions: Bookmarklets run with the permissions of the current page only. Extensions can request broader permissions that persist across all websites.
  • Persistence: Bookmarklets run only when you explicitly click them. Extensions can run automatically, in the background, or on page load.
  • Cross-browser: Bookmarklets work across different browsers with minimal or no modification. Extensions are usually browser-specific and need separate versions for each browser.
  • Updates: Bookmarklets need to be manually updated by replacing the bookmark. Extensions can auto-update through the browser's extension system.
  • Privacy: Bookmarklets don't have access to your browsing history or data unless you're on a page. Extensions can request access to browsing data, history, and more.

Bookmarklets are ideal for quick, on-demand tasks, while extensions are better for persistent functionality that needs to run automatically.

What browsers are supported?

Bookmarklets work in most modern browsers including:

  • Google Chrome: Full support, recommended version 90+
  • Mozilla Firefox: Full support, recommended version 88+
  • Safari: Full support, recommended version 14+
  • Microsoft Edge: Full support, recommended version 90+
  • Opera: Full support, recommended version 76+
  • Brave: Full support (Chromium-based)
  • Vivaldi: Full support (Chromium-based)

Some bookmarklets may have browser-specific behavior, which is usually noted in the bookmarklet description. We recommend using the latest version of your browser for the best experience. Older browsers may have limited JavaScript support that could affect bookmarklet functionality.

Do bookmarklets work on mobile?

Most bookmarklets are designed for desktop browsers. Mobile browser support varies significantly:

  • iOS Safari: Limited support. You can add bookmarklets, but accessing them requires multiple steps through the share menu.
  • Android Chrome: Better support than iOS, but still limited. Bookmarklets can be added and accessed through the bookmarks menu.
  • Mobile Firefox: Similar to Android Chrome, with better bookmarklet support than iOS.

The experience is optimized for desktop use. Mobile browsers often have security restrictions and UI limitations that make bookmarklets less convenient. Some bookmarklets may not work at all on mobile due to touch interface differences or mobile browser limitations.

  2. Navigate to our Browse page and find the updated bookmarklet
  3. Install the new version using the same method you used originally
  4. Test the updated bookmarklet to ensure it works correctly

We recommend checking for updates periodically, especially for bookmarklets you use frequently. Updated versions may include bug fixes, new features, or compatibility improvements.

How do I organize my bookmarklets?

Organizing bookmarklets helps you find and use them more efficiently:

  • Create folders: Most browsers allow you to create bookmark folders. Group related bookmarklets together (e.g., "Productivity", "Design", "Development")
  • Use descriptive names: When saving bookmarklets, give them clear, descriptive names so you can identify them quickly
  • Prioritize frequently used: Keep your most-used bookmarklets at the top of your bookmarks bar or in easily accessible folders
  • Separate by category: Organize by function (editing, viewing, testing) or by website type (social media, development, design)
  • Use browser extensions: Some browser extensions can help manage and organize bookmarks more effectively

Good organization becomes especially important as you install more bookmarklets. A well-organized bookmarklet collection can significantly improve your workflow.

Usage & Features

How do I use a bookmarklet?

For comprehensive usage instructions, see our Using Bookmarklets section.

Basic usage:

  1. Navigate to any webpage where you want to use the bookmarklet
  2. Click the bookmarklet in your bookmarks bar
  3. The bookmarklet will execute its function on the current page
  4. Some bookmarklets show results immediately, while others may require additional interaction

Remember: bookmarklets need to run on a webpage. They won't work on blank tabs or browser settings pages.

What do the status labels mean?

Each bookmarklet has a status label that indicates its reliability and testing level:

  • Stable: Fully tested and reliable. Safe to use on any website. These bookmarklets have been thoroughly tested and are unlikely to cause issues.
  • Beta: Mostly stable but may have minor issues. Generally safe but may have edge cases or minor bugs. Good for most use cases but test on non-critical pages first.
  • Experimental: New or untested features. May have unexpected behavior or bugs. Use with caution and test thoroughly before relying on it.
  • Risky: May break websites or cause unexpected behavior. These bookmarklets perform operations that could potentially interfere with website functionality. Always review the code and test on non-critical pages first.

Always check the status before using a bookmarklet, especially on important websites. The status helps you make informed decisions about which bookmarklets to use and when.

Can I run multiple bookmarklets on the same page?

Yes, you can run multiple bookmarklets on the same page, but there are important considerations:

  • Interactions: Bookmarklets may interact with each other in unexpected ways. Some are designed to work together, while others might conflict.
  • Order matters: The order in which you run bookmarklets can affect the results. Try different orders if you experience issues.
  • Testing: Test combinations on non-critical pages first to understand how they interact
  • Conflicts: If bookmarklets modify the same elements or use conflicting methods, you may experience issues

For more information on advanced usage, see our Advanced Topics section.

Troubleshooting

Why isn't a bookmarklet working?

For comprehensive troubleshooting steps, see our Troubleshooting section.

Common issues and solutions:

  • Not on a webpage: Bookmarklets need a page to run on. Make sure you're on an actual webpage, not a blank tab or browser settings page.
  • Website compatibility: Some bookmarklets may not work on certain websites due to how the site is structured or security restrictions.
  • JavaScript disabled: Ensure JavaScript is enabled in your browser settings. Bookmarklets require JavaScript to function.
  • Browser console errors: Open your browser's developer tools (F12) and check the console for error messages that might indicate what's wrong.
  • Outdated bookmarklet: The bookmarklet may have been updated. Try reinstalling the latest version from our Browse page.
  • Browser compatibility: Some bookmarklets may have browser-specific behavior. Check if the bookmarklet description mentions browser requirements.

If the problem persists after trying these steps, check the bookmarklet's status and consider reporting the issue.

How do I check for errors in the browser console?

The browser console is a powerful tool for debugging bookmarklet issues:

  1. Open developer tools by pressing F12 (Windows/Linux) or Cmd+Option+I (Mac)
  2. Click on the "Console" tab
  3. Run your bookmarklet
  4. Look for any red error messages or warnings
  5. Take a screenshot or copy the error message if you need to report the issue

Error messages in the console can provide valuable information about what went wrong, such as missing elements, JavaScript errors, or security restrictions.

Safety & Security

Are bookmarklets safe?

All bookmarklets in this collection are reviewed for safety, but you should always exercise caution:

  • Review the code: Especially for bookmarklets marked as "Risky", review the source code to understand what it does before using it
  • Trusted sources: Only use bookmarklets from trusted sources. Our collection is curated and reviewed, but always verify if you're unsure
  • Understand functionality: Read the description and understand what a bookmarklet does before installing it
  • Test first: Test bookmarklets on non-critical pages before using them on important websites
  • Check status: Pay attention to status labels (Stable, Beta, Experimental, Risky) to understand the risk level

Bookmarklets run JavaScript code on the current page, so they have access to the page's content and can modify it. Always review code from untrusted sources, and never use bookmarklets on sensitive sites unless you fully understand what they do.

Are bookmarklets safe for sensitive websites?

While we review all bookmarklets for safety, we strongly recommend being cautious when using bookmarklets on sensitive websites:

  • Banking sites: Avoid using bookmarklets on banking or financial websites. These sites have strict security measures that bookmarklets might trigger.
  • Healthcare sites: Be very cautious on healthcare or medical websites due to privacy regulations and security requirements.
  • Government sites: Government websites often have strict security policies that may conflict with bookmarklet functionality.
  • E-commerce: Be careful on shopping sites, especially during checkout processes. Bookmarklets might interfere with payment processing.

Some bookmarklets may interact with page content in ways that could trigger security warnings or interfere with site functionality. Always review the bookmarklet code if you're concerned about security, and never use bookmarklets from untrusted sources on sensitive sites.

Best practice: If you're unsure whether a bookmarklet is safe for a particular site, don't use it. It's better to be cautious than to risk security issues.

Can bookmarklets access my personal data?

Bookmarklets can only access data on the current webpage where they're running. They cannot:

  • Access your browsing history
  • Access data from other tabs or websites
  • Access files on your computer (unless you explicitly upload them)
  • Access your passwords or saved login information
  • Run in the background or when you're not using them

However, bookmarklets can access and modify the content of the current page, including any data visible on that page. This is why it's important to only use bookmarklets from trusted sources and to review the code if you're concerned about privacy.

Technical Questions

How do bookmarklets work technically?

Bookmarklets work by storing JavaScript code as a bookmark URL. When you click the bookmark:

  1. The browser executes the JavaScript code stored in the bookmark URL
  2. The code runs in the context of the current webpage
  3. The code can access and modify the page's DOM (Document Object Model)
  4. The code can interact with the page's JavaScript, CSS, and content
  5. The code runs with the same permissions as the current page

Bookmarklets are essentially JavaScript code prefixed with javascript: instead of http:// or https://. This tells the browser to execute the code rather than navigate to a URL.

For more technical details on creating bookmarklets, see our Advanced Topics section.

Why do some bookmarklets not work on certain websites?

Several factors can prevent bookmarklets from working on specific websites:

  • Content Security Policy (CSP): Some websites use strict CSP headers that prevent inline JavaScript execution, which can block bookmarklets
  • Frame restrictions: Websites in iframes or with frame restrictions may prevent bookmarklet execution
  • HTTPS/HTTP mixed content: Security restrictions may prevent bookmarklets from working on certain page types
  • Dynamic content: Websites that heavily modify their DOM after page load may interfere with bookmarklet functionality
  • Single Page Applications (SPAs): Modern web apps that use client-side routing may require bookmarklets to be run at specific times
  • Browser extensions: Some browser extensions can interfere with bookmarklet execution

If a bookmarklet doesn't work on a specific site, it's often due to security restrictions or the site's architecture. Try refreshing the page or running the bookmarklet at a different point in the page load process.

Can bookmarklets access external resources or APIs?

Bookmarklets can make requests to external resources, but they're subject to the same restrictions as the current webpage:

  • CORS (Cross-Origin Resource Sharing): Bookmarklets are subject to CORS policies. They can only make requests to APIs that allow cross-origin requests from the current domain
  • Same-origin policy: Bookmarklets run with the permissions of the current page, so they're limited by the same security policies
  • API keys: If an API requires authentication or API keys, the bookmarklet would need to include them (which has security implications)
  • HTTPS requirements: Modern browsers require HTTPS for many API requests, which can limit what bookmarklets can access

Most bookmarklets work with the current page's content rather than making external API calls, as this is more reliable and doesn't require API keys or authentication.

How are bookmarklets different from userscripts?

Bookmarklets and userscripts (like those used with Tampermonkey or Greasemonkey) are similar but have key differences:

  • Installation: Bookmarklets are stored as bookmarks and run on-demand. Userscripts are installed through browser extensions and can run automatically
  • Persistence: Bookmarklets run only when clicked. Userscripts can run automatically on page load or at specific times
  • Scope: Bookmarklets work on any page where you click them. Userscripts can be configured to run on specific domains or URL patterns
  • Storage: Userscripts can use browser storage APIs to save data between sessions. Bookmarklets typically don't have persistent storage
  • Complexity: Userscripts can be more complex and include libraries, while bookmarklets are typically simpler and self-contained

Bookmarklets are better for quick, on-demand tasks, while userscripts are better for persistent functionality that needs to run automatically.

Advanced Topics

Ready to take your bookmarklet usage to the next level? Learn about advanced techniques, creating your own bookmarklets, debugging, optimization, and best practices.

Creating Custom Bookmarklets

Basic Structure

All bookmarklets start with javascript: followed by your code. The code must be URL-encoded and typically wrapped in an IIFE (Immediately Invoked Function Expression) to avoid variable conflicts.

javascript:(function(){
  /* your code */
})();

Variable Isolation

Always wrap your code in an IIFE to prevent variable name collisions with the page's JavaScript. This creates a new scope and prevents your variables from interfering with the page.

(function() {
  var myVar = 'value';
  /* code */
})();

DOM Manipulation

Bookmarklets can access and modify the page's DOM using standard JavaScript methods like document.querySelector(), document.createElement(), and element.style. Always check if elements exist before manipulating them to avoid errors.

var el = document.querySelector('#id');
if (el) {
  el.style.color = 'red';
}

Error Handling

Always include error handling in your bookmarklets. Use try-catch blocks to handle unexpected errors gracefully. This prevents the bookmarklet from breaking and provides user feedback.

try {
  /* code */
} catch(e) {
  alert('Error: ' + e.message);
}

URL Encoding

When creating bookmarklets, you need to URL-encode special characters. Spaces become %20, quotes become %22, etc. Use online tools or JavaScript's encodeURIComponent() function. However, most browsers handle basic encoding automatically when you create bookmarks.

Code Size Limitations

Bookmarklets have practical size limits (typically 2KB-8KB depending on browser). Keep your code concise. For complex functionality, consider loading external scripts dynamically or breaking functionality into multiple bookmarklets that work together.

Advanced Techniques

Loading External Scripts

You can dynamically load external JavaScript libraries. Create a script element, set its source, and append it to the document head. This allows you to use jQuery, lodash, or other libraries in your bookmarklets.

var s = document.createElement('script');
s.src = 'https://example.com/lib.js';
document.head.appendChild(s);

User Input & Prompts

Use prompt() for user input and confirm() for yes/no questions. For better UX, create custom modal dialogs using DOM manipulation instead of browser prompts.

var input = prompt('Enter value:', 'default');
var confirmed = confirm('Continue?');

LocalStorage & SessionStorage

Bookmarklets can use browser storage to remember settings or state between page loads. Be mindful of storage limits and privacy.

localStorage.setItem('key', 'value');
var value = localStorage.getItem('key');

AJAX & Fetch Requests

Bookmarklets can make HTTP requests using fetch() or XMLHttpRequest. However, they're subject to CORS policies. You can only make requests to APIs that allow cross-origin requests from the current domain.

fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data));

Event Listeners

You can add event listeners to page elements. This is useful for creating interactive bookmarklets that respond to user actions on the page.

element.addEventListener('click', function(e) {
  // handler code
});

Timing & Delays

Use setTimeout() and setInterval() for delayed execution. Useful for waiting for dynamic content to load or creating animations.

setTimeout(function() {
  // code to execute
}, 1000);

Debugging & Testing

Console Logging

Use console.log(), console.error(), and console.warn() for debugging. Open browser developer tools (F12) to see output. Remove or comment out console statements in production bookmarklets to reduce size.

Testing Strategy

Test bookmarklets on multiple browsers (Chrome, Firefox, Safari, Edge). Test on different website types (static pages, SPAs, dynamic content). Test edge cases: empty pages, pages with errors, pages with conflicting JavaScript. Always test on non-critical pages first.

Error Detection

Check for element existence before manipulation. Validate user input. Check for required APIs or features before using them. Use feature detection rather than browser detection.

if (document.querySelector('#element')) {
  // safe to use
}

Best Practices & Optimization

Code Minification

Minify your code to reduce size: remove whitespace, shorten variable names, use single-character variable names where appropriate. Tools like UglifyJS can help, but be careful not to break functionality.

Documentation

Document your bookmarklet clearly: what it does, what it requires, known limitations, browser compatibility. Include comments in your code explaining complex logic. Write a clear description for users.

Security Considerations

Never include API keys or sensitive data in bookmarklets. Be cautious with user data. Don't collect or transmit personal information without user consent. Review code for potential security vulnerabilities before sharing.

Performance

Keep bookmarklets fast and lightweight. Avoid blocking operations. Use efficient DOM queries (cache selectors when possible). Minimize DOM manipulations. Consider using requestAnimationFrame for animations.

Code Reusability

Create reusable functions and modules. Break complex bookmarklets into smaller, testable functions. This makes code easier to maintain, debug, and share with others.

Version Control

Keep track of bookmarklet versions. Document changes, improvements, and bug fixes. Use semantic versioning if possible. This helps users know when to update and what has changed.
