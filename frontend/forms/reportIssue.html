Before Reporting

Before submitting a new issue, please take a moment to check if your problem has already been reported or if there's a solution available. When you're ready, use the report form at the top of this page to submit your issue.

Check Existing Issues

Before reporting, check if someone has already reported a similar problem. You can view existing reports to see if your issue matches any that are already being tracked.

Check Help & Support

Many common issues and questions are answered in our Help & Support page. Check the FAQ section and troubleshooting guide before reporting.

Verify the Issue

Make sure you can reproduce the issue consistently. Try refreshing the page, clearing your cache, or testing in a different browser to rule out temporary problems.

Gather Information

Collect relevant details about your issue: browser version, operating system, steps to reproduce, error messages, and screenshots. This helps us fix issues faster.

Reporting Bookmarklet Issues

If you encounter problems with a specific bookmarklet, follow these steps to report the issue effectively. You can also report a bookmarklet issue using the form at the top of this page.

Where to Report

On the Website

If the bookmarklet has a detail page, use the "Report Bug" button on that page. This automatically includes the bookmarklet name in your report.

GitHub Issues

You can view and track existing issues on our GitHub Issues page to see what's already been reported.

Discord Community

For quick questions or to discuss issues with other users, visit our Discord server.

Email Support

For sensitive issues or if you prefer email, use our Contact page to send us a message directly. Fill out the form and we'll get back to you.

What to Report

Bookmarklet Not Working

If a bookmarklet doesn't execute, produces errors, or fails to perform its intended function. Include the bookmarklet name, what you expected it to do, and what actually happened.

Browser Compatibility Issues

If a bookmarklet works in some browsers but not others, or behaves differently across browsers. Specify which browsers work and which don't.

Website-Specific Problems

If a bookmarklet works on some websites but not others. Include the URL of the problematic website and any error messages from the browser console.

Performance Issues

If a bookmarklet is slow, causes the page to freeze, or uses excessive resources. Describe the performance problem and when it occurs.

Reporting Website Issues

Issues with the website itself (navigation, display, functionality, broken links, etc.) should be reported separately from bookmarklet issues. You can report a website issue using the form at the top of this page.

Common Website Issues

Broken Links

Links that don't work, redirect incorrectly, or lead to 404 pages. Include the page where you found the link and where it should go.

Display Problems

Layout issues, text overflow, misaligned elements, or responsive design problems. Specify your screen size and browser.

Functionality Issues

Buttons that don't work, forms that don't submit, search that doesn't function, or features that are broken. Describe what should happen vs. what actually happens.

Security Concerns

Security vulnerabilities, privacy issues, or suspicious behavior. Please report these responsibly and avoid publicly disclosing security issues.

Performance Problems

Slow page loads, unresponsive pages, or excessive resource usage. Include information about your connection speed and device.

Content Errors

Typos, incorrect information, outdated content, or missing documentation. Specify the page and section where the error appears.

What Information to Include

Providing complete and accurate information helps us understand and fix issues faster. Here's what to include in your report.

Essential Information

1

Clear Description

What happened? Describe the issue in detail. What did you expect to happen? What actually happened? Be specific and include any error messages you saw.

2

Steps to Reproduce

How can we recreate it? Provide step-by-step instructions to reproduce the issue. Number each step clearly. If the issue is intermittent, note that and describe when it occurs.

3

Browser & System Info

Environment details: Include your browser name and version (e.g., Chrome 120.0.6099.129), operating system (e.g., Windows 11, macOS 14.2, Ubuntu 22.04), and device type (desktop, mobile, tablet).

4

Screenshots or Screen Recordings

Visual evidence: Screenshots help us understand the issue quickly. Include screenshots of error messages, broken layouts, or unexpected behavior. For complex interactions, a short screen recording can be invaluable.

5

Console Errors

Error messages: Open your browser's developer console (F12) and check for any error messages. Copy and paste relevant errors, warnings, or stack traces. This is especially important for JavaScript-related issues.

6

Additional Context

Extra information: Include any other relevant details: URLs where the issue occurs, whether it happens consistently or intermittently, what you were trying to accomplish, and any workarounds you've found.

For Bookmarklet Issues Specifically

Bookmarklet Name

Clearly state which bookmarklet is having the issue. Include the exact name as it appears on the website.

Website URL

Include the URL of the website where you're using the bookmarklet. Some bookmarklets are designed for specific sites.

Expected vs. Actual

Describe what the bookmarklet should do and what it actually does. Include any error messages or unexpected behavior.

Timing & Frequency

Note when the issue occurs (immediately, after delay, randomly) and how often (always, sometimes, rarely). This helps identify patterns and root causes.

Issue Categories

Understanding different types of issues helps us prioritize and route them to the right team members. Choose the appropriate category when reporting.

Bug Report

Something isn't working as expected. Use this for broken functionality, errors, crashes, or behavior that doesn't match the description. Include steps to reproduce and expected vs. actual results.

Feature Request

Suggest a new feature or improvement. Use this to propose new bookmarklets, website features, or enhancements to existing functionality. Explain the use case and benefits.

Documentation

Report documentation errors or improvements. Use this for typos, unclear instructions, missing information, or suggestions to improve our documentation and help pages.

Security

Report security vulnerabilities responsibly. Use this for security issues, privacy concerns, or vulnerabilities. Please report these privately and avoid public disclosure until they're fixed.

Performance

Report performance issues. Use this for slow loading, high resource usage, or optimization opportunities. Include performance metrics if available.

UI/UX

Report design or usability issues. Use this for layout problems, accessibility issues, confusing interfaces, or design improvements. Include screenshots showing the problem.

How to Report

Follow these steps to submit your issue report effectively. The more information you provide, the faster we can address it.

1

Choose Your Reporting Method

Website: Use the "Report Bug" button on bookmarklet detail pages for bookmarklet-specific issues, or use the report form at the top of this page. Discord: For quick questions or community discussion, visit our Discord server.

2

Select the Appropriate Issue Type

When using the report form, choose whether you're reporting a website problem or a bookmarklet issue. This helps us route your report to the right team.

3

Write a Clear Title

Use a descriptive title that summarizes the issue. For bookmarklet issues, include the bookmarklet name. Examples: "Dark Mode Toggle not working on GitHub" or "Broken link on Browse page - Category filter".

4

Fill Out All Required Information

Provide a detailed description, steps to reproduce, expected vs. actual behavior, browser and OS information, screenshots, and any error messages. The more complete your report, the faster we can fix it.

5

Provide Complete Information

Make sure to fill out all required fields in the form, including the issue type, description, and any steps to reproduce. This helps us categorize and prioritize issues.

6

Submit and Follow Up

Submit your issue through the form and be patient. We review all reports, but it may take time depending on the complexity. We'll get back to you via email if we need additional information.
