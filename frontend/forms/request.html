Before Requesting

Before submitting a new request, please take a moment to check if your idea has already been suggested or if there's a similar feature or bookmarklet available. When you're ready, use the request form at the top of this page to submit your request.

Check Existing Requests

Before requesting, check if someone has already suggested a similar feature or bookmarklet. You can view existing requests to see if your idea matches any that are already being considered.

Browse Existing Features

Explore the website and browse our bookmarklet collection to see what's already available. Your idea might already exist in a different form.

Think About Use Cases

Consider who would benefit from your request and how it would be used. The more clearly you can explain the use case, the better we can evaluate and implement your request.

Check Feasibility

While we welcome all ideas, some requests may be more feasible than others. Consider technical constraints, browser limitations, and the scope of the request when formulating your idea.

Requesting Site Features

Site features improve the website's functionality, user experience, and overall usability. You can also request a site feature using the form at the top of this page.

Types of Site Features

Navigation & UI

Improvements to site navigation, layout, menus, search functionality, or overall user interface design.

User Accounts

Features related to user accounts, profiles, authentication, preferences, or personalization options.

Bookmarklet Management

Features for organizing, categorizing, filtering, or managing bookmarklets on the site.

Performance

Improvements to site speed, loading times, optimization, or resource usage.

What Makes a Good Feature Request

Clear Problem Statement

Explain what problem the feature would solve or what need it would address. Be specific about the current limitations or pain points.

Detailed Description

Provide a clear, detailed description of how the feature would work, where it would appear, and how users would interact with it.

Use Cases

Describe specific scenarios where the feature would be useful. Include examples of how different users might benefit from it.

Requesting Bookmarklets

Bookmarklets are small JavaScript programs that run in your browser to add functionality or modify web pages. You can also request a bookmarklet using the form at the top of this page.

Types of Bookmarklets

Developer Tools

Debugging, inspection, and testing tools for developers.

Productivity

Calculators, timers, notes, and text manipulation tools.

Privacy & Security

Data clearing, bypass tools, and privacy helpers.

Automation

Auto-clicking, scrolling, and form filling tools.

Utilities

Miscellaneous helpful tools and utilities.

Visual Effects

Screen effects, animations, and visual modifications.

Games

Interactive games and entertainment to play on any website.

Hacks

Educational platform mods and website workarounds.

What Makes a Good Bookmarklet Request

Clear Purpose

Explain what the bookmarklet should do and what problem it solves. Be specific about the functionality you want.

Use Cases

Describe specific scenarios where the bookmarklet would be useful. Include examples of websites or situations where it would help.

Technical Considerations

If you have technical knowledge, mention any browser compatibility requirements, security considerations, or implementation details.

What to Include in Your Request

Providing complete and detailed information helps us understand your request better and increases the chances of it being implemented. Make sure to include the following information when submitting your request.

Required Information

Clear Title

A concise, descriptive title that summarizes your request. For features, describe what you want. For bookmarklets, describe what it should do.

Detailed Description

Explain your request in detail. What should it do? How should it work? What problem does it solve? The more detail, the better.

Use Cases

Describe specific scenarios where your request would be useful. Who would use it? When would they use it? What benefit would it provide?

Category

Select the appropriate category for your request. This helps us organize and prioritize requests more effectively.

Optional but Helpful

Visual Mockups

If requesting a UI feature, sketches or mockups can help illustrate your vision. These don't need to be professional—simple drawings work too.

Examples

Links to similar features or bookmarklets on other websites can help us understand what you're looking for.

Priority

If your request addresses a critical need or would benefit many users, mention this in your description.

Request Guidelines

Following these guidelines helps ensure your request is clear, actionable, and has the best chance of being implemented.

Be Specific

Avoid vague requests like "make it better" or "add more features." Instead, describe exactly what you want and how it should work.

Focus on Functionality

Explain what the request should do, not just what it should look like. Functionality is more important than aesthetics.

Consider Browser Compatibility

If requesting a bookmarklet, consider that it should work across different browsers. Mention any specific browser requirements if necessary.

Be Respectful

Remember that implementing requests takes time and effort. Be patient and respectful in your communication. All requests are considered, but not all can be implemented immediately.

One Request Per Submission

Submit one request at a time. If you have multiple ideas, create separate requests for each. This makes it easier to track, discuss, and implement individual requests.

Request Process

Understanding how requests are processed helps set expectations and ensures you know what to expect after submitting your request.

1

Submission

After you submit your request through the form, you'll receive a confirmation. Your request is then added to our queue for review.

2

Review

We review each request to assess its feasibility, alignment with our goals, and potential impact. This may take some time depending on the volume of requests.

3

Discussion

If we need clarification or have questions about your request, we may reach out via email. We may also discuss the request with the community to gather feedback.

4

Implementation

If approved, your request will be added to our development queue. Implementation time varies based on complexity, priority, and current workload.

5

Notification

Once implemented, you'll be notified via email (if provided) and the feature or bookmarklet will be available on the site.

Note: Not all requests can be implemented. Some may be declined due to technical limitations, scope, or alignment with project goals. However, all requests are carefully considered, and we appreciate every suggestion.
